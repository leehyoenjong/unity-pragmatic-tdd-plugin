# Alpha 단계 규칙

이 프로젝트가 Alpha 단계일 때 자동 적용됩니다.

## 핵심 원칙

> "제대로 구축하라 - SOLID + TDD"

## 필수 사항

- ✅ SOLID 원칙 적용 (특히 OCP)
- ✅ 확장 포인트 설계
- ✅ 테스트 작성 (40-60%)
- ✅ 인터페이스 분리

## 주의 사항

- ⚠️ 확장 가능한 구조 설계
- ⚠️ Beta에서 수정 어려운 부분 우선
- ⚠️ 리팩토링 적극 수행

## 기본 설정

| 항목 | 값 |
|-----|---|
| 기본 깊이 | Standard |
| TDD 비율 | 40-60% |
| 체크포인트 | 활성화 |
| Metis 상담 | 복잡한 시스템 시 |
| Momus 검증 | 선택적 |

## 자동 행동

1. 계획 시 SOLID 검토 포함
2. 인터페이스 먼저 설계
3. 테스트 먼저 작성 (TDD)
4. 병렬 구현 파이프라인 사용
