# Prototype 단계 규칙

이 프로젝트가 Prototype 단계일 때 자동 적용됩니다.

## 핵심 원칙

> "빠른 검증, 최소 복잡도"

## 허용 사항

- ✅ 간단한 구현 우선
- ✅ 테스트 최소화 (핵심 로직만)
- ✅ 하드코딩 허용
- ✅ 빠른 반복

## 주의 사항

- ⚠️ 복잡한 추상화 지양
- ⚠️ 과도한 인터페이스 분리 지양
- ⚠️ SOLID 원칙 선택적 적용

## 기본 설정

| 항목 | 값 |
|-----|---|
| 기본 깊이 | Quick |
| TDD 비율 | ~10% |
| 체크포인트 | 자동 스킵 |
| Metis 상담 | 비활성화 |
| Momus 검증 | 비활성화 |

## 자동 행동

1. 계획 질문 최소화 (1-2개)
2. 빠른 구현 진행
3. 시각적 확인 우선
