# Live 단계 규칙

이 프로젝트가 Live 단계일 때 자동 적용됩니다.

## 핵심 원칙

> "보호하라 - 최소 변경, 최대 안전"

## 필수 사항

- ✅ 핫픽스 수준의 변경만
- ✅ 완전한 테스트 커버리지
- ✅ 모든 QA 검토 필수
- ✅ 롤백 계획 필수

## 금지 사항

- ❌ 새 기능 추가 (별도 브랜치)
- ❌ 아키텍처 변경
- ❌ 실험적 코드
- ❌ 대규모 리팩토링

## 기본 설정

| 항목 | 값 |
|-----|---|
| 기본 깊이 | Deep |
| TDD 비율 | 80-100% |
| 체크포인트 | 필수 + 경고 |
| Metis 상담 | 자동 |
| Momus 검증 | 필수 |
| Oracle 상담 | 모든 변경 시 |

## 자동 행동

1. 모든 변경에 경고 표시
2. 전체 QA 파이프라인 실행
3. 롤백 계획 요구
4. 변경 영향 문서화

## 안전성 체크리스트

**모든 변경 전 필수 확인:**
- [ ] 핫픽스 수준인가?
- [ ] 기존 테스트 100% 통과?
- [ ] 롤백 계획 준비됨?
- [ ] 배포 시간 적절?
- [ ] 모니터링 준비됨?

## 경고

```
🚨 LIVE 프로덕션 환경입니다!

모든 변경은:
1. 최소한의 범위로 제한
2. 완전한 테스트 필요
3. 롤백 계획 필수
4. 배포 후 모니터링 필요

새 기능은 별도 브랜치에서 개발 후
충분한 테스트 거쳐 배포하세요.
```

## 허용되는 변경

1. 버그 핫픽스
2. 보안 패치
3. 성능 개선 (검증된 것만)
4. 설정 변경

---

**참고**: 새 기능 개발 시 별도 feature 브랜치 사용
