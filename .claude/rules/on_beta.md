# Beta 단계 규칙

이 프로젝트가 Beta 단계일 때 자동 적용됩니다.

## 핵심 원칙

> "망가뜨리지 마라 - 확장만, 수정 최소화"

## 필수 사항

- ✅ 기존 확장 포인트 활용
- ✅ 회귀 테스트 필수
- ✅ 안전성 검토 필수
- ✅ 코드 리뷰 철저

## 금지 사항

- ❌ 새 인터페이스 생성 최소화
- ❌ 기존 API 변경
- ❌ 대규모 리팩토링
- ❌ 검증되지 않은 패턴

## 기본 설정

| 항목 | 값 |
|-----|---|
| 기본 깊이 | Deep |
| TDD 비율 | 60-80% |
| 체크포인트 | 필수 |
| Metis 상담 | 자동 |
| Momus 검증 | 자동 |

## 자동 행동

1. 모든 변경에 영향 분석
2. 회귀 테스트 실행
3. QA 에이전트 검토
4. 안전성 평가 필수

## 안전성 체크리스트

모든 변경 전:
- [ ] 기존 테스트 통과?
- [ ] 영향받는 시스템 파악?
- [ ] 롤백 계획 있음?
- [ ] 성능 영향 분석?

## 경고

```
⚠️ Beta 단계에서 대규모 변경은 위험합니다.
정말 필요한 경우 Oracle 상담을 권장합니다.
```
