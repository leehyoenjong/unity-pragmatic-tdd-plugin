# 카테고리 시스템 (Category System)

Oh My OpenCode의 카테고리 시스템을 구현한 규칙입니다.

## 핵심 원칙

> **작업 유형에 따라 최적화된 접근 방식 선택**

각 카테고리는 다른 깊이, 속도, 검증 수준을 가집니다.

---

## 카테고리 목록

| 카테고리 | 설명 | 깊이 | 속도 |
|---------|------|------|------|
| `quick` | 빠른 작업, 단순 수정 | 얕음 | 빠름 |
| `standard` | 일반 작업 | 중간 | 중간 |
| `deep` | 심층 분석 필요 | 깊음 | 느림 |
| `ultrabrain` | 최대 사고력 필요 | 최대 | 가변 |
| `visual` | 시각적 요소 중심 | 중간 | 빠름 |
| `tdd` | TDD 중심 작업 | 깊음 | 중간 |

---

## 카테고리별 상세

### quick
```markdown
## Quick 카테고리

용도: 단순 수정, 오타 수정, 작은 버그 수정

특성:
- 최소 확인
- 빠른 실행
- 간결한 출력

적용:
- 한 파일 수정
- 명확한 변경
- 위험도 낮음
```

### standard
```markdown
## Standard 카테고리

용도: 일반적인 기능 구현, 중간 규모 작업

특성:
- 적절한 확인
- 체크포인트 사용
- 표준 출력

적용:
- 여러 파일 수정
- 새 기능 추가
- 중간 위험도
```

### deep
```markdown
## Deep 카테고리

용도: 복잡한 시스템, 아키텍처 변경

특성:
- 철저한 분석
- Metis/Momus 검토
- 상세한 출력

적용:
- 시스템 설계
- 아키텍처 변경
- 높은 위험도
```

### ultrabrain
```markdown
## Ultrabrain 카테고리

용도: 최대 사고력이 필요한 복잡한 문제

특성:
- 최대 깊이 분석
- Oracle 상담 활용
- 트레이드오프 분석

적용:
- 아키텍처 결정
- 성능 최적화
- 복잡한 디버깅
```

### visual
```markdown
## Visual 카테고리

용도: UI/UX, 시각적 요소

특성:
- 테스트 최소화
- 빠른 반복
- 시각적 확인 위주

적용:
- UI 구현
- 애니메이션
- 이펙트
```

### tdd
```markdown
## TDD 카테고리

용도: 테스트 주도 개발

특성:
- 테스트 먼저
- 철저한 검증
- 리팩토링 포함

적용:
- 비즈니스 로직
- 데이터 처리
- 핵심 시스템
```

## 카테고리 선택

### 자동 감지
```
요청 분석 → 키워드/패턴 매칭 → 카테고리 결정

키워드 매핑:
- "빠르게", "간단히" → quick
- "구현해줘", "만들어줘" → standard
- "분석해줘", "설계해줘" → deep
- "최선의 방법", "트레이드오프" → ultrabrain
- "UI", "화면", "애니메이션" → visual
- "TDD", "테스트 먼저" → tdd
```

### 명시적 지정
```
"quick: 오타 수정해줘"
"deep: 인벤토리 시스템 설계"
"ultrabrain: 성능 문제 분석"
```

---

## 카테고리별 워크플로우

### quick
```
요청 → 분석 → 즉시 실행 → 완료
```

### standard
```
요청 → Planner(Quick) → 구현 → 검토 → 완료
```

### deep
```
요청 → Planner(Standard) → Metis → 구현 → 검토 → 완료
```

### ultrabrain
```
요청 → Planner(Deep) → Metis → Momus → Oracle → 구현 → 검토 → 완료
```

---

## 카테고리와 프로젝트 단계 조합

| 단계 | quick | standard | deep | ultrabrain |
|-----|-------|----------|------|------------|
| Prototype | ✅ 권장 | ✅ | ⚠️ 과도 | ❌ 불필요 |
| Alpha | ✅ | ✅ 권장 | ✅ | ✅ |
| Beta | ✅ | ✅ | ✅ 권장 | ✅ |
| Live | ✅ | ⚠️ | ✅ 권장 | ✅ 권장 |

## 출력 형식

### 카테고리 감지 시
```markdown
---
📂 **카테고리: standard**

이 작업은 'standard' 카테고리로 분류되었습니다.
- 적절한 검토 수행
- 체크포인트 사용
- 표준 워크플로우

다른 카테고리를 원하시면 말씀해주세요.
---
```

### 카테고리 변경
```
"deep으로 변경해줘"
"더 꼼꼼히 해줘" → deep 전환
"빠르게 해줘" → quick 전환
```
