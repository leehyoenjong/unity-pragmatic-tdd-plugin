# 코드의 기술적 버그 가능성을 분석합니다.

## 지시사항

1. **qa-tech 에이전트를 호출**하여 버그 패턴을 분석합니다.

2. 6가지 버그 패턴 체크:

| 패턴 | 확인 항목 |
|------|----------|
| 경계값 | 0, 음수, 최대값 처리 |
| Null 참조 | null 체크 누락 |
| 동시성 | 빠른 연속 클릭, 중복 요청 |
| 상태 전이 | 사망/스턴 상태에서 행동 |
| 수학/계산 | 정수 나눗셈, 오버플로우 |
| 컬렉션 | 순회 중 수정, 인덱스 범위 |

3. Unity 특화 체크:
   - Destroy 후 접근
   - Awake/Start 순서 의존
   - 코루틴 중복 실행
   - Update에서 매 프레임 할당

4. 출력 형식:

```markdown
## 기술 분석 리포트

### 발견된 이슈
#### [Critical] {{이슈 제목}}
- 파일: {{파일}}:{{라인}}
- 패턴: {{버그 패턴}}
- 위험: {{발생 가능한 문제}}
- 수정 제안: {{코드}}
```

5. 위험도 분류:
   - **Critical**: 데이터 손실, 크래시
   - **High**: 게임 진행 불가
   - **Medium**: 불편하지만 진행 가능
   - **Low**: 사소한 문제

6. 상세 가이드: `.claude/agents/qa-tech.md`
