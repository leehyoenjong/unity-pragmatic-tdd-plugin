# Version 8.0 - Oh My OpenCode 완전 이식

**릴리스 날짜**: 2026-01-29

## 개요

Oh My OpenCode의 핵심 기능들을 Unity Pragmatic TDD Plugin에 완전히 이식한 메이저 업데이트입니다.
"똑똑하게 느껴지는" 개발 경험을 제공하기 위한 고급 자동화 기능들이 추가되었습니다.

---

## 주요 개선 사항

### 1. 작업 노트 시스템 (Work Notes System)

**문제점**: 기존에는 세션이 끝나면 작업 컨텍스트가 사라졌습니다.

**해결책**: 영구 노트 저장 시스템 도입

```
.claude/
├── notes/      # 영구 저장 노트 (세션 간 유지)
├── drafts/     # 작업 중인 문서 (계획, 설계)
├── notepads/   # 임시 메모 (에이전트 간 통신)
└── rules/      # 조건부 규칙 파일
```

**장점**:
- 세션 간 컨텍스트 유지
- 중요 결정 사항 자동 기록
- 에이전트 간 정보 공유
- 이전 작업 빠른 복원

---

### 2. TODO Continuation Enforcer

**문제점**: 작업 중간에 세션이 끊기면 미완료 작업이 방치됩니다.

**해결책**: 미완료 작업 자동 감지 및 계속 진행

```
응답 완료 전 체크:
1. Task 목록에 pending 없음?
2. 현재 응답에 미완료 TODO 없음?
3. 코드에 TODO/FIXME 주석 없음?
4. 체크리스트 모두 완료?

→ 하나라도 미완료 시 자동 계속
```

**장점**:
- 작업 완료율 100% 보장
- 사용자 개입 최소화
- 누락된 작업 방지

---

### 3. Ultrawork 모드

**문제점**: 반복적인 확인 질문으로 작업 속도 저하

**해결책**: 최대 성능 모드 도입

| 항목 | 일반 모드 | Ultrawork 모드 |
|-----|----------|---------------|
| 확인 질문 | 자주 | 최소화 |
| 병렬 처리 | 선택적 | 최대화 |
| 체크포인트 | 확인 필요 | 자동 통과 |
| 에러 복구 | 보수적 | 즉시 자동 |

**활성화**: `/eee_ultrawork` 또는 "ultrawork" 키워드

**장점**:
- 작업 속도 3-5배 향상
- 불필요한 대화 제거
- 완전 자율 실행

---

### 4. Ralph Loop

**문제점**: 복잡한 작업에서 수동 반복 필요

**해결책**: "DONE"까지 자동 반복 시스템

```
작업 시작 → 수행 → 완료?
           ↓ No    ↓ Yes
        다음 반복   "DONE"
```

**활성화**: `/eee_ralph` 또는 "ralph loop" 키워드

**장점**:
- 완전 자동화
- 최대 10회 반복
- 중단/재개 가능

---

### 5. 카테고리 시스템

**문제점**: 모든 작업에 동일한 깊이 적용

**해결책**: 작업 유형별 최적화된 접근

| 카테고리 | 설명 | 깊이 | 속도 |
|---------|------|------|------|
| `quick` | 단순 수정 | 얕음 | 빠름 |
| `standard` | 일반 작업 | 중간 | 중간 |
| `deep` | 심층 분석 | 깊음 | 느림 |
| `ultrabrain` | 최대 사고력 | 최대 | 가변 |
| `visual` | 시각적 요소 | 중간 | 빠름 |
| `tdd` | TDD 중심 | 깊음 | 중간 |

**장점**:
- 작업별 최적 리소스 할당
- 불필요한 오버헤드 제거
- 사용자 의도에 맞는 응답

---

### 6. Junior 에이전트

**문제점**: 간단한 작업에도 무거운 에이전트 사용

**해결책**: 경량 작업 전용 에이전트 (haiku 모델)

**담당 작업**:
- 빈 파일 생성
- 폴더 구조 생성
- 템플릿 적용
- 단순 수정

**장점**:
- 컨텍스트 절약 (메인 에이전트 보호)
- 빠른 응답
- 비용 효율적

---

### 7. 신규 슬래시 명령어

| 명령어 | 용도 |
|-------|------|
| `/eee_start-work` | 작업 세션 시작, 이전 세션 복원 |
| `/eee_init-deep` | 심층 분석 모드, 전체 코드베이스 분석 |
| `/eee_ultrawork` | 최대 성능 모드 활성화 |
| `/eee_ralph` | 자동 반복 모드 |
| `/eee_notes` | 노트 관리 (확인/생성/검색) |

## 아키텍처 개선

### 이전 (v7.0)
```
사용자 → 메인 LLM → 에이전트 → 결과
        (모든 판단)
```

### 현재 (v8.0)
```
사용자 → 메인 LLM → 규칙 시스템 → 에이전트
        (조율만)    (자동 판단)   (전문 실행)
                        ↓
                   노트 시스템
                   (영구 저장)
```
