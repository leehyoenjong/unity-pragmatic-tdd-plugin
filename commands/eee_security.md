# 게임 보안 취약점과 어뷰징 가능성을 분석합니다.

## 지시사항

1. **qa-security 에이전트를 호출**하여 분석합니다.

2. 주요 어뷰징 패턴 체크:

| 패턴 | 위험도 | 확인 항목 |
|------|--------|----------|
| 재화 복사 | Critical | 트랜잭션, 동시 요청 |
| 아이템 복제 | Critical | 고유 ID, 소유권 검증 |
| 패킷 조작 | High | 서버 계산, 클라이언트 신뢰 |
| 타이밍 어뷰징 | High | 서버 시간, 쿨타임 |
| 매크로/봇 | Medium | 입력 패턴, 행동 분석 |
| 메모리 조작 | Medium | 값 암호화, 체크섬 |

3. 필수 원칙:
   - **클라이언트를 믿지 마라** (서버 검증 필수)
   - 모든 재화 변경은 서버에서만
   - 트랜잭션 처리 필수

4. 출력 형식:

```markdown
## 보안 분석 리포트

### 취약점 목록
#### [Critical] {{취약점}}
- 위치: {{파일}}:{{라인}}
- 공격 시나리오: {{시나리오}}
- 영향: {{영향}}
- 수정 방안: {{코드}}
```

5. 어뷰징 테스트 시나리오 제공:
   - 동시 거래 요청
   - 결제 중 강제 종료
   - 음수 수량 요청
   - 패킷 리플레이

6. 상세 가이드: `.claude/agents/qa-security.md`
