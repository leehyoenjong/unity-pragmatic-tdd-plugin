# 단계 전환

프로젝트 단계를 전환합니다.

## 지시사항

1. 현재 PROJECT_CONTEXT.md를 읽어 현재 단계를 확인하세요.

2. 전환 유형에 따른 절차:

### Prototype → Alpha
- 프로토타입 코드 분석 (파일 수, 줄 수, 결합도)
- 리팩토링 옵션 제시:
  1. 전체 리팩토링 (권장)
  2. 핵심만 리팩토링
  3. 리팩토링 미루기 (비권장)
- 선택 후 PROJECT_CONTEXT.md 업데이트

### Alpha → Beta
- 구조 잠금 경고
- 테스트 커버리지 확인 (60%+ 권장)
- 최종 구조 리뷰 권장
- PROJECT_CONTEXT.md 업데이트

### Beta → Live
- 전체 회귀 테스트 권장
- 핫픽스 프로세스 준비 확인
- PROJECT_CONTEXT.md 업데이트

3. 전환 완료 후 새 단계의 정책 안내:

| 단계 | 리팩토링 | TDD | SOLID |
|------|----------|-----|-------|
| Prototype | 거의 없음 | ~10% | 최소 |
| Alpha | 적극적 | 40-60% | 확장 시스템에 적용 |
| Beta | 버그 수정만 | 60-80% | 기존 구조 활용 |
| Live | 거의 없음 | 80-100% | 확장 포인트만 |
