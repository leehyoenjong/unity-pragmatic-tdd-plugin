# SOLID Review

현재 코드에 SOLID 원칙을 검토하고 적용합니다.

## 지시사항

1. PROJECT_CONTEXT.md에서 현재 단계를 확인하세요:
   - **Prototype**: SOLID 최소 적용
   - **Alpha**: 확장 가능 시스템에 적극 적용
   - **Beta/Live**: 기존 구조 활용

2. OCP (Open-Closed Principle) 우선 검토:
   - 확장 포인트가 필요한 시스템 식별
   - `IStatModifier`, `ISkill`, `IItem` 등 인터페이스 설계
   - Beta에서 안전하게 기능 추가 가능하도록 구조화

3. 검토 대상 시스템:
   - 스탯/능력치 시스템
   - 스킬/액션 시스템
   - 아이템/장비 시스템
   - 버프/디버프 시스템

4. 과도한 추상화 경고:
   - 2개 이상 구현체 예상 시에만 인터페이스 생성
   - "나중에 필요할지도"는 충분한 이유가 아님
   - YAGNI 원칙 준수
